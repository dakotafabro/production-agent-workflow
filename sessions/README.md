# Sessions

Optional session tracking for practitioners who want to measure their agent-assisted workflow over time.

## What Goes Here

- Daily session logs (what was accomplished, what was learned)
- Token spend tracking (if you want to measure efficiency)
- Weekly reflections (patterns, growth edges, surprises)

## Why Track Sessions

Session tracking is optional but powerful. Over weeks, it reveals:
- Which types of work benefit most from agent assistance
- Where your workflow has friction (repeated CI failures, scope creep)
- How your collaboration depth changes as you encode more conventions
- What your actual output looks like (shipped artifacts, not just hours spent)

## Getting Started

If you want to track sessions, start simple:

```markdown
# 2026-06-29 - [Topic]

## What was built or learned
- [list]

## What pattern was followed or established
- [list]

## Next step
- [one thing]
```

You can add structure later. The habit of logging matters more than the format.

## Measurement Tools

If you use Goose, the [rp-why skill](https://github.com/block/agent-skills/tree/main/rp-why) can measure your collaboration depth automatically using the Three Dimensions model (Depth of Knowledge, Tool Maturity, Agentic Delegation Trust).
