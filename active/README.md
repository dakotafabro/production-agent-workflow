# Active

Current work-in-progress lives here. Each file represents an idea, design doc, or initiative that's in flight.

## Naming Convention

```
active/
├── feature-name-design-doc.md
├── bug-investigation-notes.md
├── prototype-exploration.md
└── team-proposal.md
```

## Lifecycle

1. Ideas start here as a one-liner (Phase 1: Capture)
2. They grow as you orient and plan (Phase 2: Orient)
3. When shipped, move to a `shipped/` directory (or delete if ephemeral)
4. When abandoned, delete (git history preserves it if needed later)

## What Belongs Here

- Design documents for upcoming work
- Investigation notes for complex bugs
- Prototype plans and exploration logs
- Proposals for team discussion
- Anything that's "in progress" but not yet code
